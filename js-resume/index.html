Campus Safe Route Guidance System

Academic Project (Aug 2013 - Dec 2013)

   This project dealt with the problem of finding the safest route between any two places in a section of map. It is intended for places like University Campus or Office locations which are big enough to be deserted most of the time but small enough to be traveled on foot or bike/skateboard.

   The underlying concept is that if there are more number of people using a specific stretch of route, it is safer to use rather than a totally deserted section of street or route.

   There are various cameras (visible and infrared) supposed to be installed at various section of the walking routes. The resulting video feed is processed to compute the average number of movements in a specific time. The more the number of movements, more safer it is. We calculate the "danger index" of each section of street and use Bellman-ford shortest path algorithm (with danger index being edge weights) to compute the safest path between two points.

   Technologies: Python, Django, OpenCV, HTML/CSS, Google Maps APIv3, jQuery, MySQL/p>
